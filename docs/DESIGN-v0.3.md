# v0.3 설계 — 스킨팩 · 여러 마리 · 키입력 반응

> 목적: v0.3의 세 기능을 구현 전에 포맷/아키텍처/보안 관점에서 확정한다.
> 우선순위: **① 스킨팩(수익화 핵심) ≫ ② 여러 마리 ≫ ③ 키입력 반응(리스크 큼, 후순위 권고)**

---

## 1. 스킨팩 포맷 `.rduck` — v0.3의 핵심(=파는 물건)

### 1.1 컨테이너
- 실체는 **zip**. 확장자는 배포용으로 **`.rduck`** 을 권장(파일연결·마켓에서 "상품"처럼 보임). 임포트 시 `.rduck`/`.zip` 둘 다 허용.
- zip 루트에 **`skin.json`(매니페스트)** + 참조 애셋(이미지/사운드).

### 1.2 매니페스트 스키마 (`skin.json`)
```json
{
  "formatVersion": 1,
  "id": "shiba-hero",                    // 고유 slug (a-z0-9-)
  "name": "시바 히어로",
  "author": "nickname",
  "version": "1.0.0",
  "character": { "image": "char.gif", "size": 130 },   // zip 내부 상대경로
  "sound":     { "file": "quack.mp3", "volume": 0.6 },  // 생략 시 합성음
  "phrases":   ["왈!", "그 변수 진짜 맞음?"],            // 생략 시 기본 문구
  "bubble":    { "textColor": "#222222", "bgColor": "#ffffff" }  // 선택
}
```
- `character.image` 필수. 나머지는 선택(생략 시 앱 기본값).
- 애셋은 매니페스트가 가리키는 상대경로로만 참조(절대경로·`..` 금지).

### 1.3 임포트 플로우
1. 설정 → 스킨 → **가져오기** → `.rduck/.zip` 선택
2. 메인 프로세스가 zip을 **메모리에서 검증**(아래 보안) → 통과 시 `userData/skins/<id>/` 에 **화이트리스트 애셋만** 추출
3. 스킨 목록에 등록(썸네일=character.image). 실패 시 사유 표시(손상/미허용 파일/용량초과)
4. 활성 스킨은 `config.activeSkin = "<id>"` 로 저장

### 1.4 설정 레이어링(활성 스킨 vs 직접 설정)
- 값 우선순위: **DEFAULTS → 활성 스킨 → (사용자가 직접 만진 필드)**
- UX 규칙: 스킨을 고르면 캐릭터/소리/문구가 그 스킨으로 채워짐. 사용자가 그 필드를 직접 바꾸면 `activeSkin=null`("직접 설정"으로 분기)로 전환 → 혼란 방지.
- 렌더러는 이미 `character/sound/phrases`를 config에서 읽으므로, **활성 스킨을 config에 머지해 내려주면 렌더러 변경 최소**(기존 duck.js 그대로 재사용).

### 1.5 보안 (⚠️ 사용자 임포트 콘텐츠 = 신뢰 불가, 최우선)
| 위협 | 방어 |
|---|---|
| **Zip Slip**(경로 탈출) | 엔트리명 정규화 후 `..`/절대경로/드라이브 문자 거부. 추출 대상은 `userData/skins/<id>/` 하위로 **resolve 검증** |
| **악성 실행/스크립트** | **확장자 allowlist만 추출**: `png,gif,apng,webp,jpg,jpeg / mp3,wav,ogg,m4a,flac / json`. 그 외(.exe/.js/.html/.dll/심링크) 전부 무시 |
| **Zip bomb** | 엔트리 수 상한(예 ≤50), **압축 해제 후 크기** 검사: 파일당 ≤10MB, 총합 ≤30MB 초과 시 중단 |
| **매니페스트 위조** | 필수필드/타입/문자열 길이 상한 검증. `image/file` 경로가 실제 추출된 allowlist 파일로 resolve되는지 확인 |
| **CSS 인젝션**(bubble 색상) | `bubble.textColor/bgColor`는 `#RRGGBB`/`rgb()` 정규식 통과값만 사용. 아니면 무시 |
| **코드 실행 원천차단** | 스킨은 **순수 애셋**. eval/HTML/JS 로드 절대 없음. 렌더러는 img.src/audio src만 세팅. 기존 CSP(`script-src 'self'`, `img/media file:`) 유지 |
- 라이브러리: zip 파싱은 검증 편한 경량 모듈(`yauzl`/`adm-zip`) 사용하되 **직접 추출 로직으로 allowlist·크기·경로 검사**를 우리 코드에서 수행(모듈 자동추출 신뢰 금지).

### 1.6 UI (설정에 "스킨" 섹션)
- 설치된 스킨 **썸네일 그리드** + 활성 표시
- **가져오기**(파일), **활성화**(선택), **삭제**(폴더 제거)
- "직접 설정" 항목(스킨 해제) + "내 스킨 만들기" 도움말 링크(포맷 문서)

## 2. 여러 마리 소환

### 2.1 아키텍처 변화
- 현재 단일 전역 `duckWin` → **`duckWins: Map<id, BrowserWindow>`** 로 일반화.
- `config.ducks = [{ id, position }]` (skin은 아래 범위 결정 참고). 앱 시작 시 저장된 수만큼 창 생성.
- IPC: `move-window/save-position/quack/move-mode` 등을 **id 기준**으로 타깃팅(현재 `duckWin` 직접 참조 → sender.webContents로 특정 창 식별).
- 트레이/우클릭 메뉴: **오리 추가 / 이 오리 제거**. 새 오리는 기존 오리 옆에 오프셋 배치.

### 2.2 범위 결정(권고)
- v0.3: **모든 오리가 활성 스킨을 공유**(단순). 개별 오리마다 다른 스킨은 v0.3.1로.
- 자동 혼잣말/부유는 창별 독립 타이머로 자연스럽게 동작(각 렌더러가 자기 스케줄).

## 3. 키입력 반응 (Bongo Cat식) — ⚠️ 리스크 큼, 후순위 권고

### 3.1 요구/제약
- "타이핑하면 오리가 반응"하려면 **전역 입력 후킹**이 필요. Electron 기본엔 없음(`globalShortcut`은 지정 조합만 감지, 전체 키 X).
- 후보: **`uiohook-napi`**(전역 키/마우스 훅, 네이티브 모듈).

### 3.2 리스크 (그래서 후순위)
- **백신 오탐**: 전역 키보드 훅 = 키로거 시그니처 → SmartScreen/AV가 더 민감. 서명 안 된 지금은 특히.
- **권한**: macOS는 손쉬운 사용 권한 필요, 네이티브 모듈 **플랫폼별 prebuild** 부담(빌드 복잡도↑).
- **프라이버시 원칙(판매 제품이라 더 중요)**: **어떤 키인지 절대 수집/저장 금지**. "입력 발생" 신호만 받아 반응, 내용·키코드 로깅 없음. 개인정보처리방침에 명시.

### 3.3 권고
- **v0.3에 넣지 말고 v0.4로**, 또는 **기본 비활성 옵트인 모듈**로 분리(원하는 사람만 설치). 코어 배포에 네이티브 훅을 기본 번들하지 않는다.

## 4. 수익화 연결 (스킨 마켓)
- **스킨팩 = 판매 단위.** `.rduck` 자체가 상품. Gumroad/itch.io에 파일 판매 → 구매자가 임포트.
- 코어 앱은 **무료·오픈소스(MIT)** 유지(성장 엔진). **유료는 프리미엄 스킨/사운드팩**(밈 캐릭터, 시즌 테마 등).
- 포맷을 **마켓 친화적으로**: 매니페스트에 id/author/version 포함, 팩은 자기완결(zip). DRM은 초기엔 없음(itch 에셋 관행).
- 로드맵: 앱 내 "스킨 갤러리"에서 무료팩 몇 개 기본 제공 + "더 받기"가 스토어로 연결.

## 5. 열린 결정 (사용자 확인 필요)
1. **키입력 반응**: v0.3에서 뺀다(권고) vs 옵트인 모듈로 넣는다 vs 지금 다 넣는다.
2. **여러 마리**: 스킨 공유(권고) vs 오리마다 개별 스킨.
3. **스킨 배포/가격**: Gumroad vs itch.io, 무료 스타터팩 + 유료팩 가격대(추후 가능).

## 6. 구현 순서 제안
1. 스킨팩 포맷 + 임포트/검증 + 활성 스킨(config 머지) + 스킨 UI  ← v0.3 메인
2. 무료 스타터 스킨 2~3개 제작(.rduck) — 포맷 검증 겸 마케팅 자산
3. 여러 마리(공유 스킨)
4. (선택) 키입력 반응 옵트인 모듈 → 실질 v0.4
