# 러버덕 디버거 (Rubber Duck Debugger) 블루프린트

## 한 줄 정의
개발자·학생이 코드가 안 풀릴 때, 바탕화면 위에 상주하는 고무오리를 클릭하면 "꽥!" 하고 반응해 주는 러버덕 디버깅 데스크탑 위젯. 이미지·문구·소리를 전부 커스텀할 수 있다.

## 가정 (사용자에게 묻지 않고 내린 결정 — 뒤집기 쉽게 명시)
- **플랫폼**: Windows 데스크탑 우선. Electron이라 mac/Linux도 코드상 지원되지만, 검증·빌드는 Windows 기준.
- **스택**: Electron (Node + Chromium). "바탕화면에 투명하게 떠서 클릭 투과되는 펫"은 Electron의 `transparent`/`alwaysOnTop`/`setIgnoreMouseEvents(forward)` 조합이 가장 표준적이고 자료가 많다. Python/PyQt로도 되지만 투명·클릭투과 처리가 더 번거로워 배제.
- **이미지는 이모지 플레이스홀더**: 기본 캐릭터는 🦆 이모지. 아이콘(트레이/앱)만 코드로 생성한 PNG(노란 오리 원형)를 넣어둠. 나중에 진짜 오리 스프라이트로 교체.
- **소리도 애셋 없이 시작**: 기본 "꽥"은 Web Audio로 실시간 합성(sawtooth+밴드패스+비브라토). 바이너리 사운드 파일 없이 즉시 소리가 난다. 사용자가 파일 지정 시 그걸로 대체.
- **설정 저장 위치**: `userData/config.json` (electron-store 같은 외부 의존성 없이 자체 JSON I/O).
- **1차 MVP는 단일 오리**: 여러 마리 동시 소환, 키입력 반응(Bongo Cat식), 애니메이션 스프라이트는 로드맵으로.

## 시장성 & 경쟁
러버덕 디버깅은 개발자 문화에 깊게 박힌 밈이고, 데스크탑펫 카테고리는 Steam에서 유료로 팔리는 검증된 시장이다(Bongo Cat 등).

| 경쟁자 | 정체 | 약점 (우리의 틈) |
|---|---|---|
| **Bongo Cat** (Steam) | 키/마우스에 반응하는 고양이 오버레이 | 러버덕 디버깅 컨셉 아님. 캐릭터/소리 자유 커스텀 약함 |
| **OpenPets** (openpets.dev) | 오픈소스 데스크탑펫 플랫폼, 플러그인·에이전트 연동 | 범용 펫. "코드 설명하며 디버깅"이라는 러버덕 정체성 없음. 무거움 |
| **Quack The Code / OSXRubberDucky / HermanFassett/rubber-duck** | 러버덕 디버깅 도구 | 창 안에 갇힌 앱 or 브라우저 확장. **바탕화면 상주 위젯이 아님** |
| **DPET (Steam)** | 커스텀 데스크탑펫 엔진 | 범용·복잡. 러버덕 니치 아님 |

**차별점**: "러버덕 디버깅"이라는 명확한 정체성 + "이미지/문구/소리 3종 완전 커스텀" + "가볍고 바탕화면에 그냥 떠있는" 위젯. 직접 겹치는 경쟁자는 사실상 없음 → 니치 선점 가능, 다만 수요는 커뮤니티 반응으로 검증 필요.

## BM 매칭
성격: **반복 사용하는 개발자용 유틸리티/토이** → 1차는 오픈소스 무료 배포로 사용자·별(star) 확보. 수익화는 **"스킨/사운드팩" 유료 판매(에셋 마켓)** + 선택적 프리미엄 티어.
- **왜 돈을 내나**: 기본 앱은 무료지만, 취향저격 캐릭터 스킨팩(시바견, 상사, 좋아하는 밈)·프리미엄 사운드팩·여러 마리 동시 소환/키반응 같은 "재미 확장"에 소액 지불. 개발자 굿즈 소비 성향과 맞음.
- **언제부터 과금**: MVP 무료 배포 → GitHub star/다운로드로 수요 검증 후, itch.io/Gumroad 또는 Steam으로 스킨팩 판매(₩2,000~5,000 단위). 코어는 계속 무료(성장 엔진), 확장이 유료.
- 2차 수익원: Steam 유료 출시(Bongo Cat 선례) 또는 "Buy me a coffee" 후원.

## 보안 체크리스트 (이 앱에 실제 해당하는 항목만)
- **유저가 파일 경로를 지정**(이미지/사운드): 파일 다이얼로그로만 경로를 받고 확장자 필터 적용. 렌더러는 `contextIsolation:true`, `nodeIntegration:false`, preload의 화이트리스트 IPC만 노출 → 렌더러에서 임의 Node 접근 차단.
- **로컬 파일 로드**: CSP 설정으로 원격 스크립트 차단(`script-src 'self'`), 이미지/미디어는 `file:`만 허용.
- **외부 네트워크 없음**: 서버·API·로그인·개인정보 수집 전부 없음 → 개인정보/인증/결제 관련 항목 해당 없음(수집 데이터 0).
- **공통**: 시크릿 하드코딩 없음, 설정은 로컬 JSON에만 저장.
> 로그인/외부 API/결제/개인정보 항목은 이 MVP 범위에 존재하지 않아 제외.

## 기술스택
- **런타임**: Electron (메인=Node, 렌더러=Chromium/HTML·CSS·JS)
- **패키징**: electron-builder (Windows nsis / mac dmg / Linux AppImage)
- **설정 저장**: 자체 `config.json` (userData) — 외부 의존성 최소화
- **소리**: Web Audio API 실시간 합성(기본) + `<audio>` 파일 재생(커스텀)
- **아이콘 애셋**: 순수 Node(zlib)로 PNG 생성하는 `scripts/gen-icons.js` → 바이너리 커밋 없이 재생성 가능
- 근거: 데스크탑 상주·투명·클릭투과·커스텀 UI가 전부 웹기술로 쉽게 해결되고, 크로스플랫폼 설치파일 배포가 판매 로드맵과 직결됨.

## MVP 범위
**IN (이번에 구현)**
- 바탕화면에 투명·프레임리스·항상위 오리 위젯 상주
- 오리 클릭 → "꽥" 소리(합성) + 말풍선("꽥!" 등 랜덤 문구) + 스퀴시 애니메이션
- 드래그로 위치 이동(이동 시 클릭으로 인식 안 함) + 위치 기억
- 오리 이외 빈 영역은 **클릭 투과**(바탕화면 아이콘 정상 클릭)
- 설정창: 캐릭터(이모지/이미지파일·크기), 말풍선 문구 목록, 소리(합성/파일·볼륨), 표시시간, 항상위 토글
- 트레이 아이콘(설정 열기 / 꽥 테스트 / 종료)
- 설정 로컬 저장·즉시 반영

**OUT (로드맵으로 미룸)**
- 애니메이션 스프라이트/걷기·표정 변화
- 키입력·마우스 반응(Bongo Cat식)
- 여러 마리 동시 소환
- 스킨/사운드팩 마켓·결제
- 자동 업데이트, 코드사이닝, Steam 출시
